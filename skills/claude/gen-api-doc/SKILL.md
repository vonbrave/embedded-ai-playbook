# 生成 C/C++ 接口 API 文档

根据指定的接口源码（头文件 + 实现文件），生成一份供人类和 AI 理解的 API 使用文档。

## 输入参数

- **$0** — 接口源码路径（头文件或源文件，或目录路径）。必填。
- **$1** — 使用该接口的示例代码路径（可选）。如提供，将从中提取实际用法作为文档示例。

## 执行步骤

### 第一步：定位源文件

1. 如果 `$0` 是目录，在该目录下查找所有 `.h` 头文件作为接口定义
2. 如果 `$0` 是具体文件（.h 或 .c/.cpp），找到同名的头文件和实现文件
3. 读取所有相关的头文件和实现文件

### 第二步：分析接口

从源码中提取：
- **不透明类型 / 句柄类型**（typedef struct ... *xxx_t 等）
- **枚举和常量定义**
- **所有公开函数**：函数签名、参数含义、返回值
- **生命周期**：创建(new/create/init) → 使用 → 销毁(deinit/uninit/destroy/free) 的调用顺序
- **内部工作机制**：从实现文件中理解每个函数实际做了什么

### 第三步：分析使用示例（如提供 $1）

从示例代码中提取：
- 实际的初始化参数和配置方式
- 调用顺序和线程模型
- 错误处理方式
- 典型的使用模式（回调 vs 轮询 vs 同步等）

### 第四步：生成文档

在接口源文件所在目录生成 `<模块名>_api.md`，包含以下章节：

```markdown
# <模块名> API 文档

<一段话概述：这个接口做什么，基于什么技术/协议>

**头文件**: `xxx.h`
**源文件**: `xxx.c`
**语言**: C / C++（是否有 extern "C"）

---

## 文档质量要求

- 参数说明要具体：写明类型、取值范围、单位、是否可为 NULL
- 生命周期要清晰：明确哪些函数必须配对调用
- 示例要可参考：尽量从实际使用代码中提取，而非凭空编写
- 注意事项要实用：标注容易出错的地方（如忘记归还缓冲区、指针失效时机）
- 使用中文撰写说明文字，代码和函数名保持原文
