### 内存管理

#### 🔴 严重 — `source/xxx.c:42`

**问题**：`malloc` 返回值未检查，直接解引用指针导致空指针崩溃。

```c
buf = malloc(size);
buf->data = 0;  // ← 若 malloc 失败则崩溃
```

**修复建议**：分配后立即检查返回值：
```c
buf = malloc(size);
if (!buf) {
    logger_error("malloc failed, size=%zu", size);
    return -ENOMEM;
}
buf->data = 0;
```
