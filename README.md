# embedded-ai-playbook

面向嵌入式 Linux / STM32 团队的 AI 工程实践仓库，用于沉淀在 Claude、Codex 等 CLI / 对话式 AI 工具中的可复用模板、skills、工作流、案例与最佳实践。

## 仓库目标

- 沉淀可复用的 Markdown 模板
- 积累 Claude / Codex CLI 的实践经验
- 形成适合嵌入式 Linux 开发的 AI 协作方法
- 支持需求分析、设计、编码、测试、评审、文档等环节
- 方便团队共享、下载、复用和持续迭代

## 目录结构

- `docs/`：说明文档、实践指南、方法论
- `skills/`：可复用的 skill 定义与使用说明
- `templates/`：需求、设计、测试、评审、Prompt 模板
- `workflows/`：Claude CLI / Codex CLI 工作流与团队协作流程
- `examples/`：嵌入式 Linux / STM32 / 测试场景示例
- `scripts/`：辅助脚本

## 建议使用方式

1. 优先复用已有模板与工作流
2. 新实践先在分支中整理，再合并到主分支
3. 通用内容优先沉淀到公共目录
4. 避免提交内部敏感信息、客户信息和私有代码

## 编码与格式规范

- 所有文本文件统一使用 UTF-8 编码
- Markdown 文件统一使用 LF 换行
- 文件命名尽量使用英文、短横线风格
- 示例内容可中文编写，但路径名建议尽量稳定、清晰
